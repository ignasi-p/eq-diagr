Text file character encoding

To make sure that text files created by this software in a given operating system mabe be used in other operating systems, text files are encoded using UTF-8 (Unicode).

In previous versions of the software (up to 2019) text files were read and written using the default character encoding of the Java Virtual Machine. This default depends on the operating system. In Ubuntu (Linux) and Mac OS text files are encoded in UTF-8 (Unicode). In Windows systems Java uses a “code-page” that is language specific.

As long as the 95 printable ASCII characters are used, a text file written in a Windows computer may be read in another machine running for example Linux, and vice versa.

However, if a text file contains non-ASCII characters, the file will be read incorrectly on a computer where Java uses a different character encoding. This situation is found if the file contains accented letters (like é or ä), or other non-ASCII characters such as the middle dot: “CaSO4·2H2O”.
